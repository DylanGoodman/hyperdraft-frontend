How Provably Fair affects Roulette

Before the game starts, a random "GameID" is generated for the following:
  1. A round "Secret" is generated by a hex string from the "GameID"
  2. An "HMAC" is generated using a "SHA512 encryption" and then converted to Hex
  3. Lastly, a random Hash is generated using the same game "Secret"
  4. After these steps, the initial "roll" happens by parsing the first 16 bytes (4 characters) of the "HMAC"
  5. The game then starts and once the game ends, we take the total of all bets and add it into the Initial Roll
  6. Once we've gotten the final roll, we then get the modulus of two numbers added, which is the winning number.

How Provably Fair affects Coinflip

Coinflip at HyperDraft is an extremely simple game from our development point to your use.

In every Coin Flip match, there are a total of 1,000 tickets between Two players. Player A (Counter-Terrorist Coin) has the odds of under 500, while Player B (Terrorist Coin) has the odds of over 500 (up to 1,000).

Before the game starts, a random "GameID" is generated for the following:
  1. A round "Secret" is generated by a hex string from the "GameID"
  2. An "HMAC" is generated using a "SHA512 encryption" and then converted to Hex
  3. Lastly, a random Hash is generated using the same game "Secret"
  4. After these steps, the initial "flip" happens by parsing the 1,000 tickets between two players
  5. The game then starts and once the game ends, the ticket has then been decided whether it has been T or CT
  6. Once we've gotten the final number, we then get the modulus of the two numbers, which is then the winning number.
